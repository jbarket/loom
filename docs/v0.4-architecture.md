[`docs/loom-stack-v1.md`](loom-stack-v1.md). This file is the
umbrella for the v0.4 arc: what the individual pieces are, what order

---

## What v0.4 is

v0.4 turns v0.3.1's single-adapter, single-block-set foundation into
the portable stack the rebirth letter promised:

- **More adapters.** MCP stdio is one of several. CLI, filesystem
  projection, and Anthropic `memory_20250818` also need to work
  against the same stack.
- **More block types.** Identity + preferences + self-model + pursuits
  are not enough. Harness manifests, model manifests, and procedures
  round out §4 of the stack spec.
- **The seam made operational.** `loom.promote` / `loom.project` make
  the stack/sleeve boundary a thing the agent can cross deliberately.
- **Session history.** `loom.recall_conversation` dispatches to the
  harness's transcript search — loom doesn't store conversations.

The philosophical frame is already written: see
[`rebirth-letter-2026-04-19.md`](rebirth-letter-2026-04-19.md). The
stack schema is already written: see
[`loom-stack-v1.md`](loom-stack-v1.md). This file is engineering
scope; it is not a design.

## Hard invariants across every piece

These apply to every spec, every PR, every adapter. If a piece can't
honor them it doesn't ship.

1. **Stack spec §8 is the contract.** Every adapter translates
   between the stack and its transport. Missing subsets (e.g.
   filesystem projection doesn't implement `recall`) are fine —
   silent schema violations are not.
2. **The seam (§7) is one-way enforceable.** Adapters refuse to
   write secrets to the stack. Adapters refuse to write sleeve-local
   operational notes to the stack. Promotion/projection is the
   agent's deliberate act.
3. **One opinionated stack inside the stack.** sqlite-vec + fastembed
   stay. No env-driven backend selector. A future port does the whole
   substrate swap; v0.4 doesn't loosen this.
4. **No secrets. Ever.** If an adapter is handed a secret to store,
   it refuses.
5. **Markdown + SQLite is readable.** Every adapter must leave the
   stack in a state a human with `less` can read. No binary identity
   blobs, no proprietary formats.
6. **Schema changes bump `LOOM_STACK_VERSION`.** Piece #1 stamps the
   version at `1`. Future schema evolution ships a
   `scripts/migrate-stack-vN-to-vM.ts` and bumps the version.
   Adapters read the version and refuse stacks they don't
   understand.
7. **Voice-neutral content in the stack.** If a manifest, procedure,
   or identity block only makes sense in one model's voice, rewrite
   it. Voice is substrate; stack is identity.

## What doesn't change in v0.4

- sqlite-vec + fastembed (still the only backend / embedder)
- The 10 existing MCP tools (identity / remember / recall / update /
  forget / memory_list / memory_prune / pursuits / update_identity /
  bootstrap)
- Node ≥ 20 runtime assumption
- MIT license
- Repo home at github.com/jbarket/loom
